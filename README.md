## Abstract

**Active-Ledger** is a federated learning framework that integrates Ethereum-based **Proof-of-Contribution (PoC)** scoring with a **1D Latent Diffusion Model (LDM)** to achieve Byzantine-robust, privacy-preserving ECG arrhythmia classification under severely non-IID data distributions.

The system addresses three fundamental challenges in clinical federated learning:

1. **Byzantine Fault Tolerance** — Malicious clients submitting poisoned model updates are detected and excluded through on-chain PoC scoring derived from immutable performance history
2. **Non-IID Class Imbalance** — Minority cardiac arrhythmia classes (LBBB, RBBB, APB, PVC) are augmented with blockchain-authorized synthetic ECG signals generated by a class-conditioned DDPM-based 1D UNet
3. **Regulatory Auditability** — Every model update, contribution score, and data generation authorization is immutably logged on an Ethereum smart contract, providing a complete provenance trail

We evaluate Active-Ledger against **6 state-of-the-art Byzantine-robust aggregation baselines** across **3 attack scenarios** (Gaussian noise, label-flip, sleeper) on the **MIT-BIH Arrhythmia Database**.

---

## Key Contributions

- **Trust-Gated Diffusion** — A novel mechanism where synthetic data generation is authorized only for clients with verified on-chain contribution history, preventing malicious data injection
- **Proof-of-Contribution (PoC) Scoring** — EMA-weighted accuracy scoring stored on-chain, enabling Byzantine-robust client selection without relying on statistical distance metrics
- **1D Latent Diffusion for ECG** — A class-conditioned DDPM UNet operating directly on 1D ECG time-series (360-sample windows) for privacy-preserving minority class augmentation
- **Comprehensive 9-Method Benchmark** — Systematic comparison against FedAvg, Krum, Multi-Krum, Coordinate-Wise Median, Trimmed Mean, and Bulyan under Gaussian, label-flip, and sleeper attacks
- **Full Provenance Pipeline** — Immutable audit trail of model hashes, accuracy reports, PoC scores, and diffusion authorization events via Solidity smart contract

### Component Overview

| Component | Module | Description |
|-----------|--------|-------------|
| **CNN-LSTM Classifier** | [`core/model.py`](core/model.py) | 3-layer 1D CNN (32→64→128) + 2-layer bidirectional LSTM with dropout — 5-class ECG classification on 360-sample windows |
| **FL Client** | [`core/client.py`](core/client.py) | Flower-compatible federated client with inverse-frequency class weighting, Byzantine attack simulation (Gaussian/label-flip/sleeper), and synthetic data augmentation |
| **PoC Server** | [`core/server.py`](core/server.py) | Server strategy implementing EMA-weighted PoC scoring from on-chain history and Top-K client selection for aggregation |
| **Blockchain Manager** | [`core/blockchain.py`](core/blockchain.py) | Python interface to the Ethereum smart contract — handles model hashing, update logging, PoC queries, and diffusion authorization |
| **1D Latent Diffusion** | [`core/diffusion.py`](core/diffusion.py) | Class-conditioned DDPM with HuggingFace UNet1DModel (2-channel input: noisy ECG + label) — trained on pooled client data, generates synthetic ECGs for minority classes |
| **Robust Aggregation** | [`core/robust_aggregation.py`](core/robust_aggregation.py) | Implementations of 6 Byzantine-robust methods: FedAvg, Krum, Multi-Krum, Coordinate-Wise Median, Trimmed Mean, Bulyan |
| **Smart Contract** | [`contracts/FLLogger.sol`](contracts/FLLogger.sol) | Solidity ^0.8.0 contract with model update logging, per-client/per-round indexing, PoC computation, and synthetic data governance |

## Quick Start

### Prerequisites

| Requirement | Version | Purpose |
|-------------|---------|---------|
| Python | ≥ 3.10 | Runtime |
| PyTorch | ≥ 2.0 | Model training |
| CUDA | ≥ 11.8 | GPU acceleration (recommended) |
| Node.js | ≥ 16 | Ganache blockchain emulator |
| Ganache | ≥ 7.0 | Local Ethereum network |

### 1. Installation

```bash
git clone https://github.com/MrRaboobi/FYP-Blockchain-FL.git
cd FYP-Blockchain-FL
pip install -r requirements.txt
npm install -g ganache
```

### 2. Data Pipeline

The data pipeline downloads, preprocesses, and partitions the MIT-BIH Arrhythmia Database:

```bash
python core/download_data.py       # Download 15 MIT-BIH records (~120 MB)
python core/preprocess_data.py     # Segment into 360-sample windows, assign 5-class labels
python core/partition_data.py      # Dirichlet non-IID split across 10 federated clients
```

### 3. Blockchain Infrastructure

```bash
# Terminal 1 — Start local Ethereum network
ganache -p 8545 --accounts 15 --deterministic --gasLimit 30000000

# Terminal 2 — Compile and deploy FLLogger.sol
python core/deploy_contract.py
```

> **Note:** Blockchain is required only for Method G (PoC-Only) and Method I (Active-Ledger Full System). Methods A–F and H run without blockchain.

### 4. Run Experiments

```bash
# Session 1: All 7 baseline methods (A–G), 40 rounds each
python main.py --mode robust-baselines

# Session 2a: Multi-Krum + Blind Diffusion (run in separate environment)
python main.py --mode session2-multikrum

# Session 2b: Active-Ledger Full System (run in separate environment)
python main.py --mode session2-active-ledger
```

## Experiments

### 9-Method Comparative Study

The evaluation compares **9 aggregation strategies** across **3 Byzantine attack scenarios**, totaling **27 experimental configurations**.

#### Session 1 — Byzantine-Robust Baselines (Methods A–G)

All 7 methods run sequentially on identical data with the same random seed (42):

| ID | Method | Type | Reference | Blockchain | Diffusion |
|:--:|--------|------|-----------|:----------:|:---------:|
| A | FedAvg | Baseline | McMahan et al., AISTATS 2017 | ✗ | ✗ |
| B | Krum | Distance-based | Blanchard et al., NeurIPS 2017 | ✗ | ✗ |
| C | Multi-Krum | Distance-based | Blanchard et al., NeurIPS 2017 | ✗ | ✗ |
| D | Median | Statistic-based | Yin et al., ICML 2018 | ✗ | ✗ |
| E | Trimmed Mean | Statistic-based | Yin et al., ICML 2018 | ✗ | ✗ |
| F | Bulyan | Hybrid | El Mhamdi et al., ICML 2018 | ✗ | ✗ |
| G | PoC-Only | **Ours (ablation)** | — | ✓ | ✗ |

#### Session 2 — Diffusion-Augmented Methods (Methods H–I)

Run in **isolated GPU environments** (separate cloud GPU sessions) to prevent cross-experiment data leakage:

| ID | Method | Type | Blockchain | Diffusion |
|:--:|--------|------|:----------:|:---------:|
| H | Multi-Krum + Blind LDM | Best baseline + blind augmentation | ✗ | ✓ (blind) |
| I | **Active-Ledger Full** | **Our complete system** | ✓ | ✓ (trust-gated) |

#### Attack Threat Models

| Attack | Adversary Behavior | # Malicious | Activation |
|--------|-------------------|:-----------:|:----------:|
| **Gaussian Noise** | Submit random Gaussian noise as model parameters | 2 / 10 | Every round |
| **Label Flip** | Flip all training labels before local training | 2 / 10 | Every round |
| **Sleeper** | Behave honestly for rounds 1–20, then submit noise | 2 / 10 | After round 20 |

### Experimental Configuration

| Parameter | Value | Rationale |
|-----------|-------|-----------|
| FL Rounds | 40 | Sufficient for convergence across all methods |
| Total Clients | 10 (8 honest + 2 Byzantine) | 20% adversary ratio — standard in Byzantine FL literature |
| Local Epochs | 3 | Balance between local overfitting and communication efficiency |
| Batch Size | 128 | Stable gradient estimates for 1D CNN-LSTM |
| Learning Rate | 0.001 | Adam optimizer default |
| Data Partition | Dirichlet (α varies) | Non-IID heterogeneity reflecting real clinical deployments |
| Loss Function | Cross-entropy with inverse-frequency class weights | Addresses 82:1 class imbalance ratio |
| Model | CNN-LSTM (32→64→128 + 2-layer BiLSTM) | Captures both morphological and temporal ECG features |
| Diffusion Pre-training | 30 epochs, DDPM, lr=1e-4 | Trained on pooled client data before FL begins |
| Diffusion Inference | 5 steps, 20 samples/class | Lightweight augmentation matched to Phase 2 parameters |
| Random Seed | 42 | Fixed across all experiments for reproducibility |

## Smart Contract

The [`FLLogger.sol`](contracts/FLLogger.sol) contract (Solidity ^0.8.0, ~300 LOC) provides four key functions:

| Function | Purpose |
|----------|---------|
| `logUpdate(round, clientId, modelHash, dataSize, accuracy)` | Immutably record a client's model update with SHA-256 hash and reported accuracy |
| `getPoC(clientId)` | Compute EMA-weighted Proof-of-Contribution score from on-chain accuracy history |
| `requestSynthetic(clientId, classLabel, quantity)` | Client requests authorization to generate synthetic data for a specific class |
| `authorizeSynthetic(requestId)` | Server authorizes the request after verifying the client's PoC score exceeds threshold |

**Events emitted:** `ModelUpdate`, `RoundCompleted`, `SyntheticRequested`, `SyntheticAuthorized`

**Gas overhead:** < 0.6 seconds per FL round on Ganache local network (negligible compared to training time).

## Dataset

**[MIT-BIH Arrhythmia Database](https://physionet.org/content/mitdb/1.0.0/)** — 48 half-hour ambulatory ECG recordings from 47 subjects, sampled at 360 Hz.

| Class | AAMI Label | Condition | Prevalence |
|:-----:|:----------:|-----------|:----------:|
| 0 | N | Normal sinus rhythm | ~82% |
| 1 | L | Left bundle branch block (LBBB) | ~8% |
| 2 | R | Right bundle branch block (RBBB) | ~7% |
| 3 | A | Atrial premature beat (APB) | ~2% |
| 4 | V | Premature ventricular contraction (PVC) | ~1% |

The extreme class imbalance (**82:1 ratio** between Normal and PVC) is the primary motivation for diffusion-based synthetic augmentation. In a federated setting with Dirichlet-partitioned non-IID data, individual clients may have **zero samples** for minority classes — making class-conditional generation essential.

## Reproducibility

All experiments are fully reproducible with the following guarantees:

- **Fixed random seed (42)** across all methods for identical weight initialization and data partitioning
- **Sequential method execution** in Session 1 with model reset between methods
- **Isolated environments** for Session 2 experiments (separate GPU sessions)
- **Checkpointed results** saved after every method and every round for crash recovery
- **Jupyter notebooks** provided in [`notebooks/`](notebooks/) for exact cloud GPU reproduction
